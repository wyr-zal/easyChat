# 2026-04-02 Excel 群发三标签分页改造计划

## 任务信息

- 任务名称：将 `excel_sender_gui.py` 改造成 3 个 TabBar 分页界面
- 开始时间：2026-04-02
- 当前状态：已完成
- 完成度：100%

## 目标

- 保持 Excel 个性化群发功能逻辑不变
- 仅调整 `excel_sender_gui.py` 的界面组织方式
- 将现有操作改成 3 个分页标签切换
- 让 Excel 数据与消息模板在同一页连续完成
- 让发送控制、预览结果、日志查看集中在同一页

## 改造范围

| 序号 | 范围 | 状态 | 完成度 | 说明 |
|------|------|------|--------|------|
| 1 | 窗口主布局重构 | 已完成 | 100% | 将纵向堆叠改为 `QTabWidget` 分页结构 |

## 分页结构

```text
Tab 1：基础设置
├─ 微信打开提示按钮
└─ 微信语言选择

Tab 2：数据与模板
├─ 文件路径 / 选择文件 / 读取数据
├─ 数据统计 / 列名概览
├─ 模板输入
└─ 占位符状态

Tab 3：发送执行
├─ 发送间隔
├─ 刷新预览
├─ 开始发送 / 停止发送
├─ 当前发送状态
├─ 发送预览表格
└─ 发送日志
```

## 实施步骤

### 阶段 1：梳理现有控件归属

- 归类现有控件到 3 个标签页
- 保留 `build_excel_group()`、`build_template_group()`、`build_preview_group()`、`build_log_group()` 的可复用内容
- 新增设置页、数据模板页和发送执行页容器

### 阶段 2：重构窗口骨架

- 将根布局从单列 `QVBoxLayout` 改成 `QTabWidget` 分页布局
- 使用 3 个固定标签页替代原先连续向下滚动的内容排列
- 保证最小窗口尺寸下仍可完整使用

### 阶段 3：重组分页内容

- 页签 1：放置微信提示按钮与语言选择
- 页签 2：放置 Excel 文件输入、读取按钮、数据统计、列名展示、模板输入、占位符状态
- 页签 3：放置发送间隔、预览、开始发送、停止发送、发送状态、预览表格、日志区域

### 阶段 4：细化交互

- 调整组标题、边距和标签页文案
- 保持发送中状态下的禁用逻辑与现状一致
- 让“刷新预览”和“开始发送”自动跳到结果页

### 阶段 5：验证

- 执行 Python 编译检查
- 手动核对窗口是否满足“3 个 TabBar 分页切换”的目标
- 检查预览、读取、开始发送、停止发送的基础流程是否仍可触发

## 验收标准

| 编号 | 验收项 | 标准 |
|------|--------|------|
| 1 | 分页展示 | 打开窗口后，顶部存在 3 个可切换标签页 |
| 2 | 功能保持 | 读取 Excel、模板输入、预览、开始发送、停止发送逻辑保持可用 |
| 3 | 分页清晰 | 各页签职责明确，数据模板和发送执行按流程分组 |
| 4 | 尺寸适配 | 在当前窗口默认尺寸下，无核心控件被遮挡 |
| 5 | 编译通过 | `excel_sender_gui.py` 改造后可通过语法编译检查 |

## 风险与应对

| 风险 | 影响 | 应对方式 |
|------|------|----------|
| 分页后用户看不到预览结果 | 影响操作连续性 | 点击预览后自动切到结果页 |
| 发送开始后用户看不到日志 | 影响发送观测 | 开始发送后自动切到结果页 |
| 分页拆分后信号连接遗漏 | 功能失效 | 尽量复用现有控件实例和槽函数，不重写业务逻辑 |
